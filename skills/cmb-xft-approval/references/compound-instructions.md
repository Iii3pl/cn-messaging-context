# 复合指令处理模式

## 场景

用户在审批列表展示后，常在一次消息中给出混合指令：

```
"1展开。2展开。3通过"
"1通过 2通过 3展开"
```

## 处理策略

### 并行执行
- 展开（review）和通过（approve）可同时发出，approve.mjs 内部自带 review 步骤
- 多条 review 可并行，多条 approve 可并行
- 不需要逐条等待

### 结果合并
- 展开结果和审批结果一起回复，避免多轮拆分
- 已通过的单据标注 ✅，展开的展示关键字段

### 金额判断捷径
- 小额（< ¥500 或打车报销）：用户说"X通过"直接执行，不必再 review
- 大额（> ¥10,000 或供应商结算/预付款）：即使用户说通过，先 review 展示关键信息再确认

## 2026-06-03 验证案例

用户输入："1 展开。2 展开。3通过"

- #1 刘祎凡（¥336 打车报销）→ review ✓
- #2 张志娟（¥31,676 供应商结算）→ review ✓

三笔并行执行，结果一次汇总回复。
