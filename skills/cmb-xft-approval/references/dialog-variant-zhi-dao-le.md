# XFT Dialog 按钮变体：「知道了」

## 发现

2026-06-25 会话实测，审批李碧合同用印（供应商）¥6,000（billId=2026062465309220）行级「通过」点击后，弹窗按钮文本为 **「知道了」**，非常用「确认」。

## 症状

```json
{"dialog_text": "提示", "buttons": ["取消", "知道了"]}
```

- 按钮：`知道了`（非 `确认`）
- 点击后：对话框关闭，审批**未提交**（row 仍在）
- 这是通知型消息框（告知信息），不是审批确认框

## 区分

| 特征 | 「确认」弹窗 | 「知道了」弹窗 |
|------|:----------:|:-----------:|
| 按钮文本 | `确认` | `知道了`、`我知道了` |
| 语义 | 二次确认审批 | 通知/信息已读 |
| 点后效果 | 审批提交，row 消失 | 仅关闭对话框 |
| 出现场景 | 正常审批流程 | SPA 通知/提示 |

## 应对

1. 点行级「通过」→ sleep 4 → `.ant-modal` 出现
2. 扫描按钮：含 `确认` → 正常流程；含 `知道了` → 通知型，可关闭
3. 关键判定不看弹窗文本，看 `stillThere`：
   - `stillThere=false` → 已批（单步生效，弹窗只是通知）
   - `stillThere=true` → 未生效，重试行级「通过」

## 样本

**李碧 ¥6,000 合同用印**（2026-06-25）：
- 第1次点「通过」→ 弹窗「知道了」→ row 仍在
- 第2次点「通过」→ 弹窗「知道了」→ 关闭 → sleep 5 → row 消失 ✅
