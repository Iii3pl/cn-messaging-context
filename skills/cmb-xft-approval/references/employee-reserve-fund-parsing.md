# 员工备用金 解析异常记录

## 症状（2026-05-31 实测）

`navigate.mjs homepage` 正确识别：

```
type: "员工备用金"
amount: "CNY 2,000.00"
subject: "5月客户投流新需求（盈领者观三平台投流以及央视文章视频投流）"
applicant: "唐诗韵"
billId: "2026052053478015"
```

但 `review.mjs --batch` 单笔解析后字段严重错误：

```
type: "员工日常报销单"    ← 错误，应为 员工备用金
amount: 300               ← 错误，应为 2000
subject: ""               ← 错误，列表有完整事由
department: null          ← 丢失
```

## 根因推测

员工备用金的详情页结构与员工日常报销单不同：
- 金额字段格式不同（可能是「总计金额(CNY)」而非「报销金额(CNY)」）
- 类型标签在 DOM 中的位置/选择器不同
- 事由字段不存在或位置偏移

与投流费用申请单类似（金额字段为「总计金额(CNY)」），但员工备用金可能还有额外差异。

## 临时处理

- 对员工备用金类型，优先信任 `navigate.mjs` 列表数据而非 `review.mjs` 详情解析
- 审批前需人工确认金额和事由
- 不要仅凭 review.mjs 输出判断

## 关联已知问题

- `extract.mjs` 的 `VALID_BILL_TYPES` 白名单已在 2026-05-31 加入「员工备用金」
- 白名单修复解决了 `pending>0 bills=[]` 的问题
- 但详情页解析适配仍需单独处理（类似投流单的专用解析路径）
