# 费用分摊明细模式 vs 汇总模式

> 发现日期：2026-05-02
> 实现日期：2026-05-02
> 来源：李思佳 ¥324.50 加班打车报销详情页实测

---

## 现象

用户从详情页复制「分摊明细」时只看到 2 行，但页面下拉后实际有 5 行。

## 根因

薪福通详情页的费用分摊区域有两个模式切换按钮：

- **汇总模式**（默认）：按 department + project 合并显示，相同组合只显示一行
- **明细模式**：展开所有分摊行，每行独立显示承担金额、比例、部门、项目

李思佳的单据：
- 汇总模式：2 行（改名了吗四组 + 规模2组）
- 明细模式：5 行（改名了吗四组×2, 项目饿组×2, 规模2组×1）

## 页面文本格式（明细模式）

```
明细模式
汇总模式                            ← 模式切换标签
费用类别    部门    项目    承担比例    承担金额(CNY)    ...
                                            ← 表头空行（tab-only）
市内交通费(项目)新                     ← category
919872092-改名了吗四组                 ← dept_id-dept_name
厦门小题旅行科技有限公司|运营中心|运营六部|改名了吗四组   ← dept_path
    视频平台代运营-B站视频代运营-20260401164841bad05-... ← project_full（前导tab）
28.5362%    92.60    89.90    2.70    ← ratio, amount, pretax, tax
-                                     ← remark

844804395-项目饿组                     ← 下一个分摊（category复用上一条）
厦门小题旅行科技有限公司|运营中心|运营六部|项目饿组
    图文平台代运营-小红书代运营-20260421170128c699f-...
15.4083%    50.00    48.54    1.46
-
...（共 5 条）
分摊合计金额：CNY 324.50
```

## 解析算法

`parseAllocations(text)` 在 `extract.mjs` 中实现：

1. `text.split(/明细模式/)[1]` → 取明细模式后的文本
2. 截取到 `分摊合计金额` 或 `审批信息`
3. 逐行状态机：
   - 费用类别行 → 新 current
   - `^\d{6,12}\s*[-–]\s*.+` → dept_id + dept_name
   - `厦门小题|...` → dept_path
   - `视频平台代运营|图文平台代运营|多平台视频代运营` → project_full + project_id + project_name
   - `[\d.]+%\s+[\d.]+\s+[\d.]+\s+[\d.]+` → ratio, amount, pretax, tax → push
4. 聚合：`aggregateBy(allocations, 'dept_id')` / `aggregateBy(allocations, 'project_id')`

## 已知边界

- 分摊行之间无显式分隔符——依赖类别行或部门行触发新 current
- 类别行仅首条出现，后续分摊复用 category
- 项目行前导可能有 `\t`，需 `replace(/^\t+/, '')` 后再匹配
- 表头列名行（部门/项目/承担金额等）需显式跳过
