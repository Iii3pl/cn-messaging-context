# XFT 主页行审批风险标签

## 背景

薪福通 XFT 主页审批列表（`ant-table-row`）中，状态列（第 24 列左右）会显示预审/风险标签，常见的值有：

| 标签 | 含义 | 是否阻断审批 |
|---|---|---|
| `已通过` | 吴亮或前置节点已完成审批 | 是（可直接跳过，但列表可能缓存残留） |
| `高风险 1项未通过` | 预审模块检测到风险项（如金额超限、CRM 项目匹配异常、项目周期已结束等） | **否** — 仅预审告警，不影响吴亮审批操作 |
| `审批中` | 当前节点在吴亮，正常待批 | 否 |

## 关键观察（2026-06-23 实战）

### 武欣荣云账户支付类

武欣荣提交的 **云账户支付** 单（`billId: 2026061662359937`, `2026061763006289`）在主页行显示 `高风险 1项未通过` 标签：

- **X1 ¥34,364**：wuLiangStatus = `已通过`（说明后端已过），但 navigate.mjs `--filter-ghosts` **未将其标记为 ghost**（仍显示 realPending）。原因：`已通过` 是预审标签，不是审批状态；审批链可能尚有后续节点，所以待审批/已审批两 tab 都包含它。
- **X2 ¥72,474**：wuLiangStatus = `审批中`，DB 曾有 agree 记录（08:43），列表仍显示 realPending。

### navigate.mjs + opencli eval 交叉验证

当 navigate.mjs 返回 `realPending=N` 但与 opencli eval rows 数差额大时，可通过主页行文本中的 `已通过` / `高风险` / `审批中` 标签判断哪些是已处理残留：

```python
# 从 rows 文本中提取状态标签
# opencli eval 输出中状态列约在第 24 字段：
# "... | 行政审批 | 已通过 | - | ..."
# "... | 行政审批 | 高风险 1项未通过 | - | ..."
# "... | 行政审批 | 审批中 | - | ..."
```

### 建议

- `高风险 1项未通过` **不阻断审批** — 只是预审告警（金额超5万/项目周期已结束/CRM承担部门不一致等），吴亮可正常通过
- 可通过主页行内 `高风险` 标签快速识别需要关注的大额单，但不需额外展开即可审批
