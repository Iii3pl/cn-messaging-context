# 审批处理工作流模式

## 吴亮的审批决策习惯

基于多轮会话观察，批处理审批时吴亮的决策模式：

### 金额分段策略

| 金额区间 | 典型动作 | 说明 |
|---------|---------|------|
| ¥0（预算单） | 展开→确认项目归属→通过 | 零金额锁编号，无资金风险但仍需确认挂哪个项目 |
| < ¥1,000 | 直接通过 | 加班打车、小额采购，无需审核详情 |
| ¥1,000 - ¥5,000 | 默认通过 | 除非类型特殊或事由不明 |
| ¥5,000 - ¥10,000 | 可能展开 | 投流费、供应商结算等中等金额会看一眼 |
| > ¥10,000 | 展开审核 | 大额供应商结算/投流/云账户支付必须复核 |

### 特殊类型处理

- **离职申请**：快速通过，不深究（非核心岗位）
- **员工备用金**：常常挂起，不急于处理
- **新单据类型**（云账户支付、供应商预付款）：展开确认结构
- **差旅报销**：大额展开，小额通过

### 跨系统切换

- 钉钉和薪福通常同时有审批，吴亮会要求"审批列表"看两边
- 钉钉审批优先处理（加班/离职/采购等简单审批快速批量通过）
- 薪福通审批涉及金额需逐笔确认

## Shell 脚本坑

### bash UID 只读变量

在 bash 脚本中 `UID` 是只读变量（当前用户 UID），赋值会导致 `UID: readonly variable` 错误。审批接口会收到空值 → `userNotExist`。

**必须使用 `USERID` 或其他变量名。**

### Token 安全过滤

Hermes 安全系统会将命令文本中出现的钉钉 token 值检测为凭证并替换为 `***`，破坏 shell 语法。必须用文件中转：

```bash
# 获取后写入文件
curl ... > /tmp/dd_token.json

# 脚本中读取
TOK=$(python3 -c "import json; print(json.load(open('/tmp/dd_token.json'))['accessToken'])")
```
