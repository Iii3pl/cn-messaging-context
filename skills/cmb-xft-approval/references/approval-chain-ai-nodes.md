# 审批链解析新节点（2026-06-29 实战）

## 现象

薪福通审批链不再只是"人名 + 状态"，开始出现非人类节点：

- **财智能体**（AI 审核）
- **用金核销**（备用金核销节点）
- **智能审核明细**（AI 摘要区域）

## 实测样本（2026-06-29 X1 汪培云 ¥1,678）

```
chain: "用金核销:审批中 | 财智能体:已通过 | 王玉晶:已通过 | 吴亮:审批中"
```

完整审批链结构：

1. 五级部门审批节点（系统自动转交）
2. 财务初审（人工）
3. 财务复审（人工）
4. CFO/副总裁审批节点
5. 吴亮（当前节点）
6. 用金核销（备用金核销 — 系统节点）
7. 财智能体（AI 审核）

## 解析注意事项

### 旧正则失效

旧的审批链正则 `/([\\u4e00-\\u9fa5]{2,4})\\s+(已通过|审批中|...)/g` 仍能匹配 "财智能体" "用金核销"（长度 4 字），但不能区分人类 vs AI 节点。

### 区分策略

```js
const HUMAN_NAMES = ['吴亮', '汪小宾', ...];  // 白名单
const isAINode = (name) => /智能体|核销|审核|系统/.test(name);
```

### 提取模板

```bash
opencli browser <sess> eval "
(function(){
  const t = document.body.innerText || '';
  const chain = [];
  const re = /([\\u4e00-\\u9fa5]{2,6})\\s+(已通过|审批中|已驳回|未开始|已完成)/g;
  let m;
  while ((m = re.exec(t)) !== null) {
    const isAI = /智能体|核销|系统|AI/.test(m[1]);
    chain.push({name: m[1], status: m[2], type: isAI ? 'system' : 'human'});
  }
  return JSON.stringify(chain, null, 2);
})()
"
```

## 业务含义

- **财智能体已通过** → 财务合规检查通过（AI 审核）
- **用金核销审批中** → 系统在核对备用金使用情况
- 这些节点**不影响吴亮的审批决策**，但报告时要标注"AI 系统节点"以免用户困惑

## 审批链展示建议

汇报时按人类/系统分组：

```
人类节点：
  - 王玉晶（部门负责人）：已通过
  - 吴亮（运营 VP）：审批中 ← 当前节点

系统节点：
  - 财智能体（AI 审核）：已通过
  - 用金核销（备用金核对）：审批中
```
